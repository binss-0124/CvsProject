supabase db 연동 방법
1. supabase에서 New project 버튼을 눌러 새 프로젝트를 생성
2.  - Project name : 자유
    - Database Password : 자유
    - Region : Northest Asia (Seoul) 선택
    - Create new project 누르기
3. 좌측 메뉴바에서 Authentication 클릭 --> 세부 메뉴에서 Sign In / Providers 클릭 --> Confirm email 비활성화 --> Save changes 버튼 클릭
    - 이거 안하면 큰일나효..
4. 좌측 메뉴바에서 Project Settings 클릭
    - 세부메뉴에서 Data API 선택 --> url 복사
    - .env 파일의 VITE_SUPABASE_URL=이부분에 붙여넣기. ( '=' 사이에 빈칸 있으면 안대염!)
    - 세부메뉴에서 API Keys 클릭 anon public 키 복사
    - .env 파일의 VITE_SUPABASE_ANON_KEY=이부분에 붙여넣기.
5. 좌측 메뉴바에서 SQL Editor 선택
    - 프로젝트 파일의 00_setup_all_advanced.sql 파일 열고 전체 복사
    - sql editor에 붙여넣고 하단의 녹색 run 버튼 누르기
6. 설정 끗.
7. 프로젝트 폴더 루트에서 터미널 열기
8. npm install
9. npm run dev

통합테스트 시나리오

### **2.1 고객 계정 생성**
1. 애플리케이션 접속
2. "로그인" --> "회원가입" 클릭
3. 고객 정보 입력:
   ```
   이메일: customer1@test.com
   비밀번호: test1234
   이름: 테스트 고객1
   역할: 고객
   ```
4. 회원가입 완료 확인
5. 자동으로 `/customer` 페이지로 리다이렉트되는지 확인

### **2.2 점주 계정 생성**
1. 로그아웃
2. "회원가입" 클릭
3. 점주 정보 입력:
   ```
   이메일: shopowner1@test.com
   비밀번호: test1234
   이름: 테스트 점주1
   역할: 점주
   지점명: 테스트 편의점
   지점 주소: 서울시 강남구 테스트로 123
   전화번호: 02-1234-5678
   ```
4. 회원가입 완료 확인
5. 자동으로 `/store` 페이지로 리다이렉트되는지 확인

### **2.3 본사 계정 생성**
1. 로그아웃
2. "회원가입" 클릭
3. 본사 정보 입력:
   ```
   이메일: hq@test.com
   비밀번호: test1234
   이름: 테스트 관리자
   역할: 본사
   ```
4. 회원가입 완료 확인
5. 자동으로 `/headquarters` 페이지로 리다이렉트되는지 확인
6. 총 지점수에 1이 뜨고, 지점별 현황에 테스트 편이점이 나오는지 확인
7. 대시보드에서 해당 테스트 편의점을 누르면 상세 정보가 나오는지 확인
8. 지점 관리, 상품 관리가 잘 나오는지 확인

## 🏪 **3단계: 점주 기능 테스트**

### **3.1 점주 로그인**
1. `shopowner1@test.com`으로 로그인
2. 점주 대시보드 접속 확인

### **3.2 지점 정보 확인**
1. 대시보드에서 지점 정보 표시 확인
2. 지점명, 주소, 전화번호 등이 올바르게 표시되는지 확인

### **3.3 상품 관리**
1. "물류 관리" 탭 클릭
2. 기본 상품들이 표시되는지 확인:
    - 농심 새우깡 90g
    - 농심 신라면 120g
    - 코카콜라 500ml
3. 해당 상품들의 현재재고가 모두 0인지 확인

### **3.4 물류 요청 **
1. 물류 요청 버튼 클릭
2. 유선순위, 희망 배송일, 요청 수량을 작성 후 요청 생성 버튼 클릭
3. 점주 계정에서 로그아웃 

### **3.5 재고 관리**
1. 본사 계정(hq@test.com)으로 로그인
2. 대시보드 중 대기중 물류요청과, 긴급 요청에 올바르게 나오는지 확인
3. 물류관리 탭으로 이동하여 요청이 잘 나오는지 확인
4. 승인 버튼 클릭 후 수량 입력하고 승인 버튼 클릭
5. 배송 시작 클릭
6. 배송 완료 클릭
7. 물류 트리거 작동 팝업 확인
8. 로그아웃

## **3.6  물류 반영 확인**
1. 점주 계정(shopowner1@test.com)으로 로그인
2. 물류 관리 탭으로 이동 후 재고가 반영되고, 물류 요청 목록에 반영되었는지 확인
3. 로그아웃

## 🛒 **4단계: 고객 기능 테스트**
1. 고객 계정(customer1@test.com)으로 로그인
2. 로그인 후 지점선택에 개설한 테스트 편의점이 나오는지 확인
3. 해당 지점 선택
4. 상품 카드들이 잘 나오고, 카테고리가 잘 작동하는지 확인
5. 원하는 상품을 장바구니에 추가하고 장바구니로 이동
6. 픽업 또는 배송을 선택 후 주문하기 클릭
7. 선택한 픽업 혹은 배송이 다음 페이지에서도 잘 유지되어 넘어왔는지 확인
8. 결제 방법으로 토스페이 선택
9. 약관 동의 후 결제 방법 선택하기 이동
10. 토스페이로 결제하기 선택
11. 결제하세용
12. 결제하면 결제 성공 페이지가 나오고 몇 초 후에 주문내역으로 이동
13. 이때 오류가 있어용 주문이 두 개가 나오는데 새로고침하면 사라져용(이유는 모름~)
14. 주문 추적을 눌러서 상세 정보 확인
15. 로그아웃

## 🏢 **5단계: 고객 주문 받기**
1. 점주 계정(shopowner1@test.com)으로 로그인
2. 대시보드에 주문 내역이 잘 나오는지 확인
3. 하단의 빠른 액션에 주문 관리 클릭
4. 주문 확인을 누르면 해당 주문이 처리중으로 넘어감
5. 처리중 탭에서 준비 시작 클릭 --> 완료처리까지 쭉 누르기
6. 모두 완료되면 완료/취소 탭으로 주문이 이동함.
7. 물류 관리 탭으로 이동하여 해당 주문의 물품 수량이 실제로 재고 수량에서 감소되었는지 확인
8. 로그아웃
