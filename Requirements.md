# Requirements Document

## Introduction

이 기능은 편의점 재고 관리 시스템에 제품별 유통기한 관리 기능을 추가하는 것입니다. products 테이블에 품목별 기본 유통기한을 설정하고, 물류 배송 완료 후 자동으로 유통기한이 감소하는 시스템을 구현합니다.

## Requirements

### Requirement 1

**User Story:** 시스템 관리자로서, 각 제품의 기본 유통기한을 설정할 수 있어야 하므로, 제품별로 적절한 유통기한 관리가 가능하다.

#### Acceptance Criteria

1. WHEN products 테이블에 새로운 제품이 추가 THEN 시스템은 shelf_life_days 필드에 기본 유통기한(일수)을 저장해야 한다
2. WHEN 제품의 기본 유통기한이 설정 THEN 시스템은 해당 값을 모든 관련 재고에 적용해야 한다
3. WHEN 제품의 기본 유통기한이 수정 THEN 시스템은 새로 입고되는 재고부터 변경된 값을 적용해야 한다

### Requirement 2

**User Story:** 시스템으로서, 물류 요청이 배송 완료되면 해당 시점을 기준으로 제품별 유통기한 카운트다운을 시작해야 하므로, 정확한 유통기한 추적이 가능하다.

#### Acceptance Criteria

1. WHEN 물류 요청 상태가 'delivered'로 변경 THEN 시스템은 해당 제품들의 delivered_at 시간을 기록해야 한다
2. WHEN 재고 조회 시 THEN 시스템은 delivered_at 시간과 현재 시간의 차이를 계산하여 남은 유통기한을 실시간으로 계산해야 한다
3. WHEN 재고의 계산된 유통기한이 0 이하 THEN 시스템은 해당 재고를 만료 상태로 표시해야 한다

### Requirement 3

**User Story:** 점주로서, 유통기한이 임박한 제품을 쉽게 식별할 수 있어야 하므로, 적시에 할인 판매나 폐기 등의 조치를 취할 수 있다.

#### Acceptance Criteria

1. WHEN 제품의 남은 유통기한이 7일 이내 THEN 시스템은 경고 상태(주황색)로 표시해야 한다
2. WHEN 제품의 남은 유통기한이 3일 이내 THEN 시스템은 위험 상태(빨간색)로 표시해야 한다
3. WHEN 제품의 유통기한이 만료 THEN 시스템은 만료 상태(회색)로 표시해야 한다
4. WHEN 점주가 필터를 적용 THEN 시스템은 유통기한 상태별로 제품을 필터링해야 한다

### Requirement 4

**User Story:** 점주로서, 유통기한 관련 통계를 확인할 수 있어야 하므로, 재고 관리 전략을 수립할 수 있다.

#### Acceptance Criteria

1. WHEN 점주가 재고관리 페이지에 접근 THEN 시스템은 유통기한 임박 제품 수를 표시해야 한다
2. WHEN 점주가 재고관리 페이지에 접근 THEN 시스템은 만료된 제품 수를 표시해야 한다
3. WHEN 점주가 재고관리 페이지에 접근 THEN 시스템은 정상 유통기한 제품 수를 표시해야 한다

### Requirement 5

**User Story:** 점주로서, 물류 요청 시 유통기한 정보를 고려할 수 있어야 하므로, 적절한 수량과 타이밍으로 발주할 수 있다.

#### Acceptance Criteria

1. WHEN 점주가 물류 요청을 생성 THEN 시스템은 현재 재고의 남은 유통기한 정보를 표시해야 한다
2. WHEN 점주가 물류 요청을 생성 THEN 시스템은 유통기한 임박 제품에 대한 권장사항을 제공해야 한다
