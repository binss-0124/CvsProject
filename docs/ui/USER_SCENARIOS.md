# 사용자 시나리오 명세서

## 🎭 사용자 시나리오 개요

편의점 종합 솔루션의 실제 사용 상황을 기반으로 한 **구체적인 사용자 시나리오**들을 정의합니다. 각 시나리오는 실제 사용자의 니즈와 행동 패턴을 반영합니다.

## 👤 고객 시나리오

### 시나리오 1: 직장인 점심 주문

**사용자**: 김대리 (29세, 직장인, 스마트폰 중급 사용자)
**상황**: 점심시간 12시, 사무실에서 편의점 도시락 주문
**목표**: 빠르고 간편하게 점심 해결

```
🕐 12:00 - 점심시간 시작
김대리가 사무실 책상에서 스마트폰으로 편의점 앱을 실행합니다.

1. 앱 실행 → 자동 로그인 (저장된 세션)
2. 홈 화면에서 "주변 편의점" 확인
   - GPS 기반으로 사무실 근처 3개 편의점 표시
   - 거리: 도보 2분, 5분, 7분
3. 가장 가까운 "GS25 을지로점" 선택
4. 카테고리에서 "도시락/김밥" 탭 선택
5. 인기 상품 "불고기덮밥"과 "아메리카노" 장바구니에 추가
6. 픽업 시간을 "12:30"으로 설정 (30분 후)
7. 카드 결제로 9,500원 결제 완료
8. 주문번호 확인 후 알림 설정

🕐 12:25 - 픽업 5분 전 알림
"주문하신 상품이 준비되었습니다. GS25 을지로점에서 픽업해주세요!"

🕐 12:30 - 편의점 방문
주문번호를 점주에게 보여주고 상품 수령 완료
```

**핵심 포인트**:
- 빠른 접근성 (2탭 이내로 주문 가능)
- 명확한 픽업 시간 설정
- 실시간 알림으로 편의성 제공

### 시나리오 2: 야식 배송 주문

**사용자**: 이학생 (22세, 대학생, 기숙사 거주)
**상황**: 밤 11시, 과제하다가 배고파서 야식 주문
**목표**: 기숙사까지 안전하게 배송받기

```
🕐 23:00 - 과제 중 배고픔
기숙사 자취방에서 라면과 음료수를 주문하려고 합니다.

1. 앱 실행 → 홈 화면에서 "배송 가능한 편의점" 필터
2. 배송 반경 내 "CU 대학로점" 선택 (24시간 운영)
3. 검색창에 "라면" 입력 → 인기 상품들 확인
4. "신라면 큰컵" 2개, "콜라 500ml" 1개 장바구니 추가
5. 배송 주소 확인 → 기숙사 상세 주소 (층, 호수)
6. 요청사항에 "문 앞에 놓고 문자 주세요" 입력
7. 쿠폰 적용 (신규회원 1,000원 할인)
8. 총 6,500원 → 5,500원 결제
9. 예상 배송시간 30-40분 확인

🕐 23:35 - 배송 도착 알림
"상품이 배송 완료되었습니다. 확인해주세요!"
문자 메시지와 함께 문 앞에서 상품 수령
```

**핵심 포인트**:
- 야간 배송 가능 매장 필터링
- 상세한 배송 주소 및 요청사항
- 할인 쿠폰 적용으로 가격 메리트

### 시나리오 3: 가족 장보기

**사용자**: 박주부 (35세, 주부, 3인 가족)
**상황**: 주말 오전, 일주일치 생필품 주문
**목표**: 무겁고 많은 상품들을 편리하게 주문

```
🕐 10:00 - 주말 장보기
집에서 일주일치 생필품을 편의점에서 주문합니다.

1. 앱 실행 → 즐겨찾는 "세븐일레븐 신도림점" 바로 접속
2. 장보기 리스트 체크
   - 생활용품: 휴지, 세제, 샴푸
   - 식품: 우유, 계란, 빵, 과자
   - 음료: 생수 2L 6개, 아이 주스
3. 카테고리별로 체계적으로 담기
   - "생활용품" → 휴지 3팩, 세제 대용량
   - "유제품" → 우유 1000ml 3개
   - "베이커리" → 식빵, 크림빵
4. 대용량 주문으로 배송료 무료 (3만원 이상)
5. 포인트 1,200점 사용
6. 총 42,000원 → 40,800원 결제
7. 배송시간 "오후 2-4시" 선택

🕐 14:30 - 배송 완료
문 앞 배송으로 무거운 생필품들을 편리하게 수령
```

**핵심 포인트**:
- 대량 주문 시 배송료 혜택
- 카테고리별 체계적 쇼핑 지원
- 포인트 적립/사용 시스템

### 시나리오 4: 새벽 근무 점주의 하루

**사용자**: 최사장 (45세, CU 강남점 점주, 5년 경력)
**상황**: 새벽 6시 출근, 하루 운영 준비
**목표**: 효율적인 매장 운영 및 매출 관리

```
🕐 06:00 - 매장 오픈 준비
점주가 매장에 도착해서 태블릿으로 점주 앱을 실행합니다.

1. 로그인 → 어제 매출 현황 확인
   - 일매출: 850,000원 (목표 대비 112%)
   - 주문 건수: 127건
   - 인기 상품: 아메리카노, 김밥, 담배
2. 재고 현황 체크
   - 품절 상품 5개 발견 (우유, 빵, 음료수)
   - 부족 상품 12개 (노란색 경고)
3. 긴급 발주 요청 생성
   - 품절 상품 우선 선택
   - 오전 배송 요청 (10시 이전)
   - 본사에 승인 요청 전송

🕐 07:00 - 출근 러시아워 시작
새벽 주문들이 들어오기 시작합니다.

4. 실시간 주문 알림 (딩동~ 소리)
   - "아메리카노 2잔, 샌드위치 1개" 주문 접수
   - 예상 준비시간 5분으로 설정
   - 상태를 "준비중"으로 변경

🕐 09:30 - 발주 상품 도착
본사에서 승인된 상품들이 배송 도착

5. 입고 처리
   - 배송된 상품들 스캔하여 재고 자동 업데이트
   - 유통기한 확인 및 진열
   - 품절 상품들 판매 재개

🕐 22:00 - 하루 마감
하루 운영을 마무리하며 정산을 진행합니다.

6. 일일 정산
   - 총 매출: 1,150,000원
   - POS와 앱 매출 대조 확인
   - 이상 거래 건수: 0건
   - 내일 준비사항 메모 작성
```

**핵심 포인트**:
- 실시간 재고 관리의 중요성
- 빠른 발주 시스템으로 품절 최소화
- 매출 분석을 통한 운영 최적화

### 시나리오 5: 신규 점주의 첫 달

**사용자**: 김신규 (28세, 새로 오픈한 GS25 점주)
**상황**: 오픈 3주차, 시스템 적응 중
**목표**: 시스템 숙달 및 안정적 운영

```
🕐 Week 3 - 시스템 학습 단계
신규 점주가 점차 시스템에 익숙해지고 있습니다.

1. 일일 루틴 확립
   - 오전 8시: 전날 매출 확인
   - 오전 9시: 재고 현황 점검
   - 점심시간: 주문 집중 관리
   - 저녁 시간: 발주 계획 수립
   - 마감 시간: 정산 및 내일 준비

2. 발주 패턴 학습
   - 초기: 수동으로 모든 상품 발주
   - 2주차: 인기 상품 패턴 파악
   - 3주차: 자동 발주 제안 시스템 활용
   - 계절 상품과 날씨 연동 주문

3. 고객 응대 개선
   - 처음: 주문 처리에 5-7분 소요
   - 현재: 평균 3분으로 단축
   - 실시간 알림 활용한 빠른 응답
   - 고객 요청사항 세심하게 처리

4. 분석 기능 활용
   - 매출 그래프로 트렌드 파악
   - 시간대별 매출 분석
   - 인기 상품 순위 확인
   - 경쟁점 대비 성과 분석

한 달 후 성과:
- 평균 일매출 30% 증가
- 고객 만족도 4.8/5.0
- 재고 회전율 최적화
- 본사 우수 점주 표창 수상
```

**핵심 포인트**:
- 학습 곡선 고려한 UI/UX 설계
- 점진적 기능 활용 가이드
- 성과 측정을 통한 동기부여

### 시나리오 6: 지역 매니저의 주간 업무

**사용자**: 이매니저 (35세, 서울 강남 지역 담당 매니저)
**상황**: 월요일 오전, 주간 업무 계획 수립
**목표**: 담당 지역 50개 매장 효율적 관리

```
🕐 Monday 09:00 - 주간 업무 시작
지역 매니저가 사무실에서 관리자 대시보드를 확인합니다.

1. 주말 성과 리뷰
   - 담당 지역 50개 매장 중
   - 우수 매장: 12개 (매출 목표 120% 이상)
   - 관심 매장: 8개 (매출 목표 80% 미만)
   - 문제 매장: 2개 (시스템 오류 또는 민원)

2. 신규 매장 승인 검토
   - 3건의 신규 매장 신청서 검토
   - 입지 분석 보고서 확인
   - 사업 계획서 및 자금 계획 검토
   - 현장 실사 일정 조율

3. 발주 요청 일괄 처리
   - 주말 밀린 발주 요청 47건
   - 긴급 배송 필요 매장 5개 우선 처리
   - 대량 발주 할인 적용 가능 상품 확인
   - 재고 부족 예상 상품 선제적 공급

🕐 Wednesday 14:00 - 중간 점검
주 중반 현황을 점검하고 필요한 조치를 취합니다.

4. 실시간 모니터링
   - 매출 부진 매장 "CU 역삼점" 원인 분석
   - 최근 1주일 매출 15% 감소 확인
   - 점주와 직접 통화하여 상황 파악
   - 마케팅 지원 및 프로모션 제안

5. 긴급 상황 대응
   - "GS25 선릉점" 시스템 오류 신고
   - 기술팀과 연계하여 즉시 해결
   - 해당 매장 매출 손실 최소화 방안 수립
   - 유사 문제 예방을 위한 시스템 점검

🕐 Friday 17:00 - 주간 마무리
한 주를 마무리하며 다음 주 계획을 수립합니다.

6. 주간 성과 보고서 작성
   - 지역 전체 매출: 목표 대비 108%
   - 신규 매장 2개 승인 완료
   - 발주 처리율: 99.2%
   - 고객 만족도: 평균 4.6/5.0

7. 다음 주 중점 관리 사항
   - 신규 매장 오픈 지원 (2개 매장)
   - 부진 매장 집중 관리 프로그램
   - 여름 성수기 대비 재고 확충
   - 점주 교육 프로그램 운영
```

**핵심 포인트**:
- 다수 매장의 통합 관리 효율성
- 데이터 기반 의사결정 지원
- 예외 상황 신속 대응 체계

### 시나리오 7: 신상품 출시 프로젝트

**사용자**: 박상품기획자 (32세, 상품기획팀 과장)
**상황**: 여름 신상품 3종 전국 출시 프로젝트
**목표**: 효과적인 신상품 런칭 및 성과 모니터링

```
🕐 Project Week 1 - 출시 준비
신상품 출시를 위한 시스템 준비 단계입니다.

1. 상품 정보 등록
   - 신상품 3종 마스터 데이터 입력
   - 상품 이미지, 영양정보, 가격 설정
   - 카테고리 분류 및 진열 순서 결정
   - 프로모션 기획 (출시 기념 할인)

2. 매장별 출시 계획
   - 1단계: 수도권 주요 매장 100개 (Week 1)
   - 2단계: 전국 주요 도시 300개 (Week 2)
   - 3단계: 전체 매장 1,000개 (Week 3)
   - 지역별 선호도 고려한 맞춤 배치

3. 점주 교육 자료 준비
   - 상품 특징 및 판매 포인트
   - 진열 가이드라인
   - 고객 응대 스크립트
   - 온라인 교육 영상 제작

🕐 Project Week 2 - 출시 실행
신상품이 실제 매장에 공급되기 시작합니다.

4. 실시간 출시 모니터링
   - 1단계 매장 100개 출고 완료
   - 재고 입고 현황 실시간 확인
   - 점주 피드백 수집 및 분석
   - 초기 판매 데이터 수집

5. 이슈 대응
   - 3개 매장에서 포장 불량 신고
   - 즉시 해당 로트 회수 조치
   - 대체 상품 긴급 공급
   - 품질팀과 협력하여 원인 분석

🕐 Project Week 4 - 성과 분석
출시 한 달 후 성과를 종합 분석합니다.

6. 매출 성과 분석
   - 상품 A: 예상 매출 대비 145% (대성공)
   - 상품 B: 예상 매출 대비 78% (저조)
   - 상품 C: 예상 매출 대비 92% (평범)
   - 지역별/매장별 편차 분석

7. 향후 전략 수립
   - 성공 상품 A: 전국 확대 및 시리즈 확장
   - 부진 상품 B: 가격 조정 및 프로모션 강화
   - 평범 상품 C: 계절성 고려한 재기획
   - 차기 신상품 개발 시 참고사항 정리
```

**핵심 포인트**:
- 단계적 출시를 통한 리스크 관리
- 실시간 모니터링 및 빠른 대응
- 데이터 기반 성과 분석 및 피드백

### 시나리오 8: 픽업 지연 상황 처리

**관련 사용자**: 고객(김대리), 점주(최사장), 본사(고객센터)
**상황**: 주문 처리 지연으로 인한 고객 불만 발생
**목표**: 신속한 문제 해결 및 고객 만족도 회복

```
🕐 12:30 - 예정된 픽업 시간
김대리가 점심 픽업을 위해 편의점에 방문했지만 주문이 준비되지 않음

[고객 관점 - 김대리]
1. 앱에서 주문 상태 확인 → 여전히 "준비중"
2. 점주에게 직접 문의 → "5분만 더 기다려달라"
3. 회사 복귀 시간이 촉박해져 스트레스 증가
4. 앱 내 "고객센터" 버튼으로 문의 등록

[점주 관점 - 최사장]
1. 아르바이트생 실수로 주문 누락 발견
2. 고객에게 사과하며 즉시 상품 준비 시작
3. 점주 앱에서 "지연 사유" 입력 및 "완료 예정시간" 업데이트
4. 고객에게 음료 한 잔 서비스 제공 결정

[본사 관점 - 고객센터]
1. 고객 문의 접수 즉시 해당 매장에 연락
2. 상황 파악 후 고객에게 진행 상황 안내
3. 보상 방안 제안 (다음 주문 시 할인쿠폰)
4. 해당 매장에 재발 방지 교육 안내

🕐 12:40 - 상황 해결
- 김대리: 주문 상품 + 서비스 음료 수령, 할인쿠폰 받음
- 최사장: 사과와 함께 재발 방지 약속, 직원 교육 실시
- 고객센터: 만족도 조사 발송, 매장 모니터링 강화

결과:
- 초기 불만 → 오히려 더 좋은 고객 경험으로 전환
- 점주의 적극적 대응이 고객 충성도 증가
- 시스템 개선으로 유사 상황 예방
```

**핵심 포인트**:
- 실시간 소통을 통한 신속한 문제 해결
- 다층적 고객 서비스 체계
- 문제를 기회로 전환하는 서비스 마인드

### 시나리오 9: 대량 주문 및 특별 요청

**관련 사용자**: 고객(회사 총무), 점주, 본사 물류팀
**상황**: 회사 워크샵용 대량 주문 및 특별 배송 요청
**목표**: 대량 주문의 원활한 처리 및 고객 만족

```
🕐 화요일 오전 - 대량 주문 요청
회사 총무가 금요일 워크샵용 간식을 대량 주문합니다.

[고객 관점 - 회사 총무]
1. 일반 주문량을 초과하는 대량 주문 (음료 100개, 과자 50개)
2. 앱에서 자동으로 "대량 주문 상담" 팝업 표시
3. 특별 요청사항 입력:
   - 배송일: 금요일 오전 9시
   - 배송지: 회사 로비
   - 요청사항: 종류별 박스 포장, 영수증 별도 발행

[점주 관점 - 편의점 사장]
1. 대량 주문 알림 수신 및 재고 확인
2. 현재 재고로는 60% 정도만 충당 가능
3. 점주 앱에서 "본사 지원 요청" 버튼 클릭
4. 부족 상품 목록 및 배송일 정보 전송

[본사 관점 - 물류팀]
1. 대량 주문 지원 요청 접수
2. 주변 매장 재고 확인 및 조달 계획 수립
3. 특별 배송 차량 및 인력 배정
4. 고객에게 주문 확정 및 세부 일정 안내

🕐 금요일 오전 9시 - 배송 완료
- 정확한 시간에 정확한 장소로 배송 완료
- 종류별 정리된 포장 및 영수증 제공
- 고객 만족도 매우 높음

🕐 사후 관리
- 총무에게 정기 주문 제안 (매월 워크샵 간식)
- 해당 점주에게 우수 고객 서비스 포상
- 대량 주문 프로세스 개선사항 피드백 수집
```

**핵심 포인트**:
- 일반 주문과 차별화된 대량 주문 처리 프로세스
- 다부서 협업을 통한 고객 요구사항 만족
- 특별 서비스를 통한 장기 고객 관계 구축
